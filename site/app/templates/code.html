Reproducible Offline Real data

Everything you see here — the Explore charts and every Data download — is produced by the thin FastAPI + DuckDB + Plotly app below, reading the real {{ counts.slice_db_h }} curated sample in app/data/freenic_slice.duckdb.

Run it, get identical output (over the slice). Every JSON API response and every download is computed deterministically from the read-only slice — no hidden state, no network call, no random seed. Install the pinned deps, run uvicorn app.main:app, and the charts and download bytes regenerate byte-for-byte over the same {{ counts.slice_db_h }} slice. (The full {{ counts.release_size_iec }} parquet release is served separately at data.freenic.org and is not part of this bundle.)

1 · Read-only DuckDB access + the JSON API

Every connection opens read_only=True; the explorer's charts fetch these endpoints, each a parameterized read over a real slice table.

app/main.py
{% filter forceescape %}{% raw %}def _con() -> duckdb.DuckDBPyConnection:
    """Read-only connection to the baked curated slice."""
    return duckdb.connect(str(DB_PATH), read_only=True)


def _rows(sql: str, params: list | None = None) -> list[dict]:
    con = _con()
    try:
        cur = con.execute(sql, params or [])
        cols = [d[0] for d in cur.description]
        return [dict(zip(cols, r)) for r in cur.fetchall()]
    finally:
        con.close()


@app.get("/api/failures_by_year")
def api_failures_by_year() -> JSONResponse:
    return JSONResponse(_rows(
        "SELECT year, n_failures, total_assets, total_loss "
        "FROM agg_failures_by_year ORDER BY year"
    )){% endraw %}{% endfilter %}

3 · The downloads (correct content-types)

The Data page serializes each real slice table to CSV and Parquet (the user-facing formats — no JSON dataset downloads). Each format carries its true media type — the known "CSV-served-as-JSON" failure is avoided by deriving the content-type from the artifact's declared format. Parquet is written natively by DuckDB COPY to a throwaway temp file (the DB stays read-only).

app/downloads.py
{% filter forceescape %}{% raw %}# CSV + Parquet only — no JSON dataset downloads (DOWNLOAD_AND_FORMATS DNA).
MEDIA = {
    "csv": "text/csv; charset=utf-8",
    "parquet": "application/vnd.apache.parquet",
    "zip": "application/zip",
}


def _parquet_bytes(table: str) -> bytes:
    """Native Parquet via DuckDB COPY to a throwaway temp file (read-only DB)."""
    fd, tmp = tempfile.mkstemp(suffix=".parquet")
    os.close(fd)
    try:
        con = _con()
        try:
            con.execute(
                f'COPY (SELECT * FROM "{table}") '
                f"TO '{Path(tmp).as_posix()}' (FORMAT PARQUET)"
            )
        finally:
            con.close()
        return Path(tmp).read_bytes()
    finally:
        try:
            os.unlink(tmp)
        except OSError:
            pass


def render_artifact(art: dict) -> tuple[bytes, str, str]:
    """Return (bytes, media_type, download_filename) for a single artifact."""
    raw = _bytes_for(art["table"], art["fmt"])
    fname = f"freenic_{art['table']}.{art['fmt']}"
    return raw, MEDIA[art["fmt"]], fname{% endraw %}{% endfilter %}

4 · Beyond the sample — the full dataset

The on-site sample is for browsing and charting. For arbitrary SQL across all 42 tables of the complete dataset, point the freenic_py / freenic_r packages — or a plain DuckDB httpfs client — at the published parquet release; only the row-groups a query touches are fetched.
