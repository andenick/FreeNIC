# FreeNIC — Open Banking Data Platform

**~4.97 billion rows of US banking regulatory data spanning 1863–2026, unified into 59 base tables (46 `main` + 6 `catalog` + 7 `dict`) plus 52 shaped views, covering 217,210 institutions.**

*(Counts verified read-only against the live warehouse — see
[`Technical/coverage_analysis/live_counts.json`](Technical/coverage_analysis/live_counts.json), the
single source of truth regenerated by `tools/freenic_live_counts.py` and gated by
`tools/check_doc_counts.py`. Exact live total: 4,965,894,682 rows. The platform grew well past the
original "~2.25B / 37-table / 20-source" framing — the largest tables are now `call_report_filings`
1.92B, `ubpr_ratios` 1.25B, and `ncua_5300` 1.18B; UBPR, NCUA credit unions, FR Y-15, HMDA, the
extended-NIC identity layer, the dictionary taxonomy layer, and the canonical `clean_bank_panel` are
all included.)*

---

## Why This Exists

Empirical banking research requires combining data from many regulatory sources — call reports, holding company filings, failure records, stress test results, deposit data — each in different formats with different identifiers. FreeNIC harmonizes these into a single relational schema with Python, R, and AI agent interfaces.

All data comes from public regulatory filings (FFIEC, FDIC, Federal Reserve, OCC) and freely available academic databases. No proprietary data, no paywalls.

## Quick Start

```bash
git clone https://github.com/andenick/FreeNIC.git
cd FreeNIC

# Python interface
cd Outputs/freenic_py
pip install -e .
```

```python
import freenic

freenic.set_data_dir("Outputs/parquet")
freenic.list_tables()
freenic.lookup_institution("jpmorgan")
freenic.get_financials(1039502, "BHCK2170")  # JPMorgan total assets
freenic.get_failures(start_year=2008, end_year=2010)
```

The Parquet data files are not included in the repo. You can either query the **hosted release** directly (no download — see below) or build them from source using the [Ingestion Pipeline](#ingestion-pipeline).

### Hosted data — query in place, no download

The full parquet release (59 files, ~13 GB, with `SHA256SUMS.txt`, `PROVENANCE.csv`, and
`DATA_LICENSE.md`) is served read-only with HTTP byte-range support at:

**<https://data.freenic.org>**

DuckDB `httpfs` fetches only the row-groups a query touches, so a typical query transfers
megabytes, not gigabytes:

```python
import duckdb

con = duckdb.connect()
con.execute("INSTALL httpfs; LOAD httpfs;")
con.execute("""
    SELECT failure_year, COUNT(*) AS failures
    FROM 'https://data.freenic.org/bank_failures.parquet'
    GROUP BY failure_year ORDER BY failure_year DESC LIMIT 10
""").df()
```

```r
library(duckdb)
con <- dbConnect(duckdb())
dbExecute(con, "INSTALL httpfs; LOAD httpfs;")
dbGetQuery(con, "SELECT COUNT(*) FROM 'https://data.freenic.org/occ_historical.parquet'")
```

Browse the file index (and download whole files) at <https://data.freenic.org/>; verify
downloads against [`SHA256SUMS.txt`](https://data.freenic.org/SHA256SUMS.txt). CORS is open,
so browser DuckDB-Wasm clients work too.

## Taxonomy companion — bank-data-dictionary

FreeNIC holds the *observations*; the **[bank-data-dictionary](https://github.com/andenick/bank-data-dictionary)**
holds the *structure*. The dictionary names and groups the raw MDRM codes by schedule and
line item, and ships an empirically-adjudicated reconciliation registry (FFIEC official
edits + `official_calc` derivations + curated identities) whose verdicts were validated
**directly against this warehouse**. FreeNIC imports it as-is into the `dict` schema
(version-pinned in `dict.meta`), generating 39 schedule-shaped views and a coverage-audit
map that guarantees every raw variable is reachable through at least one view. Browse the
dictionary at **<https://andenick.github.io/bank-data-dictionary>**; see
[`Outputs/TAXONOMY.md`](Outputs/TAXONOMY.md) for the layer reference and refresh procedure,
and [`Outputs/RELEASE_NOTES_v1.3.md`](Outputs/RELEASE_NOTES_v1.3.md) for the dictionary-sync
release.

## Ingestion Pipeline

The ingestion pipeline transforms raw regulatory data into the unified Parquet schema. Scripts are numbered and run in order:

| Script | What it does |
|--------|-------------|
| 00 | Setup: create database schema |
| 01 | MDRM variable dictionary |
| 02 | NIC institution attributes |
| 03 | CRSP-FRB bank identifier link |
| 04–05 | BHCF Y-9C filings (TXT + CSV formats) |
| 06 | Y-9C schema verification |
| 07 | Chicago Fed call reports (1976–2002) |
| 08 | Luck Historical Database |
| 09 | OCC historical (1867–1904) |
| 10 | Build institution catalog |
| 11 | Build convenience views |
| 12 | Export to Parquet |
| 13 | Validate all tables |
| 16–19 | FDIC (failures, financials, SOD) |
| 20 | Cross-source identifier crosswalks |
| 23–25 | DFAST, Pillar 3, FDIC history |
| 27 | Fed H.8 aggregate series |
| 28–30 | Failing Banks panel, bank-identifier catalogs, stress scenarios |
| 31–33 | SDI feature panel + FFIEC CDR fair-value/unrealized-loss panel |
| 45 | **Clean bank panel** (canonical from-raw nominal+real $ levels; fixes robin_panel's uncalibrated absolutes) |

## Setup

Querying the published Parquet/DuckDB outputs needs no configuration. To **run the
ingestion pipeline yourself**, point it at where it should read inputs and write
derived panels via two environment variables:

| Variable | Purpose | Default |
|----------|---------|---------|
| `DATA_ROOT` | Where externally-sourced inputs not bundled under `Inputs/` are read (e.g. FFIEC CDR bulk ZIPs, the Failing Banks panel, bank-identifier catalogs, stress scenarios) | `./data` |
| `OUTPUT_ROOT` | Where derived feature panels (`*.parquet`) are written | `./outputs` |

```bash
git clone https://github.com/andenick/FreeNIC.git && cd FreeNIC
pip install -r requirements.txt
# optional: only needed to re-run ingestion
export DATA_ROOT=/path/to/your/data
export OUTPUT_ROOT=/path/to/your/outputs
```

Each ingestion script first looks for its source under `Inputs/<source>/`; if that
is absent it falls back to `$DATA_ROOT/<source>/`. See **`data/MANIFEST.md`** for the
exact files each script expects and where to download them.

### API keys — bring your own

Querying needs no keys. Some ingestion sources are public APIs that offer an optional
free key for higher rate limits:

- **FRED** (Fed H.8 / banking series): free key at
  <https://fred.stlouisfed.org/docs/api/api_key.html>; set `FRED_API_KEY`.

Copy `.env.example` to `.env` and fill in any keys you use. The FDIC BankFind, FFIEC
CDR, and Chicago Fed downloads require no key.
