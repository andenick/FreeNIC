What FreeNIC is

Empirical banking research means stitching together call reports, holding-company filings, failure records, stress-test results, and deposit data — each in a different format with a different identifier. FreeNIC harmonizes these public regulatory sources (FFIEC, FDIC, Federal Reserve, OCC, FRED, NCUA, SEC) plus freely available academic databases into a single relational schema with Python, R, and AI-agent (MCP) read layers. No proprietary data, no paywalls; the code is MIT and the data terms travel per source.

FreeNIC is its own standalone project in the heterodata domainspace: the bulk / SQL layer for US banking-regulatory microdata. The full warehouse runs to {{ counts.base_rows_h }} observations across {{ counts.base_tables }} base tables from {{ counts.source_datasets }} public datasets, covering ~217,000 institutions (banks, thrifts, holding companies, and credit unions) from 1863 to 2026 — including three billion-row fact panels (call reports, UBPR ratios, NCUA 5300 credit-union filings).

Query the full release remotely — no {{ counts.release_size_iec }} download

The full parquet release is served read-only with HTTP byte-range support at https://data.freenic.org/ (live). DuckDB reads parquet directly over HTTP via httpfs, fetching only the row-groups a query needs:

DuckDB · remote httpfs
{% filter forceescape %}{% raw %}INSTALL httpfs; LOAD httpfs;

-- the CC0 historical OCC layer, 1863-1941, queried in place
SELECT year, COUNT(*) AS n_banks
FROM 'https://data.freenic.org/occ_historical.parquet'
GROUP BY year ORDER BY year;

-- join two remote tables without downloading either in full
SELECT bf.bank_name, bf.failure_year, bf.total_assets
FROM 'https://data.freenic.org/bank_failures.parquet' bf
ORDER BY bf.total_assets DESC
LIMIT 10;{% endraw %}{% endfilter %}

The same files install locally for the pip/R packages: pip install freenic → freenic.set_data_dir("https://data.freenic.org"). See API for the full access matrix and Data for the per-file catalog with direct URLs.

The historical layer is public domain (CC0)

FreeNIC carries the finhist / OCC historical layer — occ_historical, bank balance sheets digitized from the primary U.S. Office of the Comptroller of the Currency Annual Report to Congress volumes, 1863–1941.

This slice is released under CC0 1.0 Universal (public-domain dedication) via the Harvard Dataverse deposit (Correia, Luck & Verner, Replication Data for: Failing Banks, DOI 10.7910/DVN/Q22XR1). Redistribution, modification, and commercial use are permitted with no legal conditions; the underlying OCC reports are U.S. federal government works in the public domain. Hosting FreeNIC therefore also satisfies "host a version of the finhist data" — it is already inside the release as occ_historical.parquet. A courtesy citation is requested but not required.

Data Provenance Record

Sources
The slice is built from the published FreeNIC parquet release — FDIC + Robin bank failures, FRED banking macro series, and the NIC active-institutions table. Those exports derive from public regulatory and cited academic sources (FFIEC, FDIC, Federal Reserve, OCC, plus freely available academic banking databases). The full {{ counts.source_datasets }}-dataset composition of the complete dataset is listed below. Public code + provenance: github.com/andenick/FreeNIC.
Construction
build_slice.py reads the offline parquet exports and writes the compact read-only DuckDB shipped here (app/data/freenic_slice.duckdb, {{ counts.slice_db_h }}). The core tables (bank_failures, fred_series) are copied verbatim; institutions_active is the real active-only subset with trimmed columns; the agg_* tables and fred_catalog are precomputed real aggregates (GROUP-BY rollups, no imputation). Units
Monetary fields (total_assets, total_deposits, estimated_loss in bank_failures) are reported in US-dollar thousands ($000s) as published by FDIC. fred_series.value carries each FRED series' own native unit (level, percent, index — see the series name in fred_catalog). Dates are calendar dates; counts (n_failures, n, n_obs) are integers.
Scope
  • Curated browsable sample. The on-site explorer ships failures, FRED banking series and active institutions (+ aggregates) — a {{ counts.slice_db_h }} curated subset of the published tables, for browsing and charting without the full corpus.
  • Complete dataset via package + API. The multi-GB harmonised panels (call reports, Y-9C, SOD, etc.) and the full {{ counts.base_rows_approx }}-row warehouse are available via the freenic_py / freenic_r packages and remote DuckDB httpfs over the parquet release — see API.
  • Active-only institutions. institutions_active is the real active subset (of 217,210 total), with a trimmed column set — not the full institution history.
  • No imputation. Aggregates are straight GROUP-BY rollups of the real rows; null source values are excluded, never invented.
Read-only access
The on-site JSON APIs that feed the explorer charts are read-only — every DuckDB connection opens read_only=True and the DB cannot be written — and every query is parameterized (? bind), never string interpolation of user input. The browsable slice covers these tables: {% for t in allowed_tables %}{{ t }}{% if not loop.last %}, {% endif %}{% endfor %}. For arbitrary SQL across all {{ counts.base_tables }} base tables, use the R/Python package or the remote httpfs pattern (see API).
Downloads
Every slice table and the full release are downloadable as CSV and Parquet (Parquet preferred for analysts; no JSON dataset downloads) from the Data page; the complete {{ counts.release_size_iec }} parquet release is also queryable remotely over DuckDB httpfs at data.freenic.org. Bulk slice: freenic_slice.zip. Release metadata co-hosted at data.freenic.org: CODEBOOK.md + per-table codebook/<table>.csv column codebooks, data_dictionary.csv, CHANGELOG.md, and CITATION.cff.
Refresh cadence
Quarterly, on FFIEC release. The FFIEC publishes new Call Report, UBPR, FR Y-9C and NCUA 5300 quarters roughly 75 days after each quarter-end; FDIC SDI and FRED follow their own quarterly/ongoing schedules. FreeNIC is refreshed by the automated freenic-refresh protocol (acquire → ingest → validate → regenerate → republish). The CC0 historical OCC layer (1863–1941) is fixed and does not refresh.
Citations
FDIC — bank failures & SDI financials (public). Board of Governors of the Federal Reserve System — FRED banking series (public). FFIEC — call reports (NIC / structure data, public). OCC — historical national-bank records (public). Robin failing-banks panel — academic compilation (cited). All sources are public regulatory data or cited academic databases; no proprietary feeds.
