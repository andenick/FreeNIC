FreeNIC is reachable four ways: the on-site JSON endpoints over the browsable sample, remote DuckDB httpfs over the full parquet release, the freenic_py / freenic_r packages, and an MCP server for AI agents. This page is the access matrix.

1 · On-site JSON endpoints (the browsable slice)

2 · DuckDB httpfs — the full release, remotely

The full parquet release is served read-only with HTTP byte-range support at https://data.freenic.org/ (live). DuckDB reads parquet over HTTP and fetches only the row-groups a query touches — so you can join multi-GB tables without downloading them.

DuckDB · remote httpfs 
{% filter forceescape %}{% raw %}import duckdb
con = duckdb.connect()
con.execute("INSTALL httpfs; LOAD httpfs;")

BASE = "https://data.freenic.org"

# count rows in a remote table (only metadata + needed row-groups fetched)
con.execute(f"SELECT COUNT(*) FROM '{BASE}/institutions.parquet'").fetchone()

# the CC0 historical OCC layer, queried in place
con.execute(f"""
    SELECT year, COUNT(*) AS n_banks
    FROM '{BASE}/occ_historical.parquet'
    GROUP BY year ORDER BY year
""").df()

# join two remote tables
con.execute(f"""
    SELECT bf.bank_name, bf.failure_year, bf.total_assets
    FROM '{BASE}/bank_failures.parquet' bf
    ORDER BY bf.total_assets DESC LIMIT 10
""").df(){% endraw %}{% endfilter %}

Verify integrity against SHA256SUMS.txt (also served at data.freenic.org) after any local mirror. See Data for the full per-file URL catalog.

3 · The freenic_py & freenic_r packages

The read layers wrap DuckDB with banking-aware helpers (institution lookup, financial time series, hierarchy, failures, variable search). Point them at a local parquet dir or the remote URL once, then query in one line.

Python · freenic 
{% filter forceescape %}{% raw %}pip install freenic        # (or: pip install -e . from the repo)

import freenic
# local mirror …
freenic.set_data_dir("/path/to/freenic/parquet")
# … or the remote release (env: FREENIC_DATA_DIR=https://data.freenic.org)
freenic.set_data_dir("https://data.freenic.org")

freenic.list_tables()
freenic.lookup_institution("jpmorgan")
freenic.get_financials(1039502, "BHCK2170")          # JPMorgan total assets
freenic.get_failures(start_year=2008, end_year=2010)
freenic.query("""
    SELECT i.name_legal, bf.failure_year, bf.total_assets
    FROM bank_failures bf
    JOIN institutions i ON bf.cert = i.fdic_cert
    WHERE bf.failure_year >= 2008
    ORDER BY bf.total_assets DESC LIMIT 10
"""){% endraw %}{% endfilter %}
R · freenic

4 · MCP server (AI agents)

An MCP (Model Context Protocol) server exposes the same read layer to AI agents — an agent can list tables, look up institutions, pull financial series, and run SQL against the configured parquet dir or remote URL. Set FREENIC_DATA_DIR and run the server from Technical/freenic_mcp/ in the repo.

shell · MCP server 
{% filter forceescape %}{% raw %}export FREENIC_DATA_DIR=https://data.freenic.org   # or a local mirror
python Technical/freenic_mcp/server.py{% endraw %}{% endfilter %}

FreeNIC and Robin

FreeNIC sits alongside Robin, the heterodata series API. The division of labour: Robin is the series API — curated, versioned individual data series with citation/provenance, fetched one series at a time; FreeNIC is the banking-data bulk / SQL layer — the full regulatory microdata warehouse you query relationally (joins across call reports, filings, failures, identity crosswalks). They are complementary: Robin for "give me this series with its lineage," FreeNIC for "let me run SQL across the whole banking panel." A future Robin↔FreeNIC bridge (FreeNIC's banking series surfaced through the Robin API) is anticipated but is not part of this release.
